# Contributing to WRXWear

Thank you for your interest in contributing to the WRXWear project! We appreciate contributions from developers, designers, and enthusiasts who share our vision of revolutionizing workwear through AI and 3D printing technology.

## How to Contribute

### Reporting Bugs
If you find a bug, please report it by [opening an issue](https://github.com/wrxwear/2026/issues) with:
- A clear, descriptive title
- A detailed description of the problem
- Steps to reproduce the issue
- Expected behavior vs. actual behavior
- Screenshots or logs (if applicable)
- Your environment (OS, Python version, etc.)

### Proposing Features
We welcome feature suggestions! Please open an issue with:
- A clear title describing the feature
- A detailed description of the proposed functionality
- Use cases and benefits
- Potential implementation approach (optional)
- Any relevant mockups or examples

### Submitting Pull Requests
1. **Fork the repository** and create a new branch for your work:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Follow our code standards**:
   - Use meaningful variable and function names
   - Include docstrings for all functions
   - Write tests for new functionality
   - Follow PEP 8 for Python code

3. **Commit your changes** with clear, descriptive messages:
   ```bash
   git commit -m "Add feature: description of changes"
   ```

4. **Push to your fork** and submit a pull request:
   ```bash
   git push origin feature/your-feature-name
   ```

5. **Provide context in your PR**:
   - Clearly describe what your PR does
   - Link to relevant issues
   - Include screenshots or examples if applicable
   - Explain any breaking changes

### Design Contributions
We need help with 3D design and CAD work! If you're interested in contributing designs:
- Please use standardized file formats (STL, STEP, or IGES)
- Include documentation of component specifications
- Provide compatibility notes with existing modules
- Test designs for modularity and durability

### Documentation
Improvements to documentation are always welcome:
- Typo fixes and clarity improvements
- Better examples and tutorials
- API documentation
- Design specifications
